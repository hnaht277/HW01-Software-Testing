# HW01 – QA/QC Jobs · 20 Defects · Test a Physical Product

## General Information
- **Exercise ID:** HW01-AI
- **Duration:** 5 hours
- **Form:** Individual Assignment
- **Submission:** Moodle (report) + GitHub repo link (artifacts)
- **AI Policy:** Open – MANDATORY declaration + attached AI Audit Report
- **Bloom-AI level:** G9.1 (Understand) -> G9.6 depending on HW

### Outcomes
1. Describe the 2026+ QA/QC job-market landscape (including AI-augmented testing roles).
2. Distinguish work that AI can replace / assist / cannot replace in QA/QC.
3. Design test cases for a SPECIFIC physical product (not a software product).
4. **Understand:** ask an AI Tool for an ISTQB-process mindmap and find 3 mistakes.
5. **Analyse:** analyse AI-generated test cases and find ≥ 3 missed edge cases.

---

## Requirements

### Requirement 1 – QA/QC Job Market 2026+ (40 pts)
- **Goal:** Find 10 QA/QC job postings PUBLISHED WITHIN 60 DAYS of the submission date.
- **Mandatory:** ≥ 3 positions REQUIRING AI/LLM/automation-AI skills.
- **Each posting must include:**
  - Link
  - Dated screenshot (must show your account name in the corner)
  - Job description
  - Required skills
  - Salary
  - "AI Impact Analysis" (1–2 sentences per posting)

### Requirement 2 – 20 Software Defects 2022–2026 (20 pts)
- **Goal:** Find 20 software defects publicized between 2022 and 2026.
- **Mandatory:** ≥ 5 defects related to AI/LLM (hallucination, prompt injection, bias).
- **Each defect must include:**
  - Source link
  - Description
  - Severity
  - Consequences
  - Solution

### Requirement 3 – Test cases for ONE physical product (40 pts)
- **Goal:** Choose a SPECIFIC household device (fan / water filter / rice cooker / smart bulb, etc.).
- **Deliverables:**
  - 1 photo of THE DEVICE + your student ID card in the SAME frame.
  - Declare brand, model, year, serial number (mask the middle 4 chars).
  - Design 15 test cases (Objective / Input / Steps / Expected / Actual / Verdict).
  - Execute and record short videos (≤ 60s) for ≥ 5 test cases and aim to find ≥ 5 defects.
  - ≥ 3 test cases must be edge cases the AI Tool could NOT find (provide screenshot of AI conversation AND written explanation of why AI missed them).

---

## AI Collaboration Protocol (Mandatory)
1. **Allowed tools:** Any AI Tool of your choice (ChatGPT, Claude, Gemini, Copilot, Cursor). Must declare what you used.
2. **AI Audit Report:** Each AI-generated artifact must have a 5-section entry:
   - (1) Prompt + tool (Full prompt + tool name + timestamp).
   - (2) AI output (Full AI output or red-bordered screenshot with annotations).
   - (3) Verdict (VALID / INVALID / INCOMPLETE with reasoning grounded in ISTQB).
   - (4) Reasoning (2–5 sentences citing slide / ISTQB).
   - (5) Student fix (Corrected artifact highlighting changes).
   - Summarize AI accuracy ratio and conclude when AI should be used/not used.
3. **AI Critique:** 200–300-word paragraph critiquing the AI (Where did it get wrong? Why? What principle learned?).
4. **Mandatory Disclosure:** Standard template included before appendices.
5. **Anti-AI-Cheat mechanisms:**
   - Device photo with student-ID card in the same frame.
   - Execution videos must contain your own voice narration.
   - All 10 job-posting screenshots must show your login / username.
   - Prompt log `.md` with timestamps for every AI prompt.
6. **Oral Defense:** Random 30% of students will be invited.
